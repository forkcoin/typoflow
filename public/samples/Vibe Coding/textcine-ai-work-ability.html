不用先相信术语

把模型的能力，变成你自己的工作能力

Harness、Agent、Skill、Vibe Coding 听起来很新。但它们要解决的麻烦，其实很普通。

真实麻烦

你说了想法，AI 做了一部分。

你继续让它改，它又改坏了别的地方；隔天回来，忘了上次做到哪一步；角色越开越多，文件和提示词反而开始互相覆盖。

问题不在词新不新。

问题在于：你有没有一套能让 AI 持续工作的安排。

四个词，先这样理解

Vibe Coding
先把脑子里还没完全说清的画面跑出来，再一边看、一边改、一边收紧。
Agent
让 AI 承担某一类责任的角色。重点不是多，而是边界。
Skill
把反复用到的一套做法，写成可调用的方法包。
Harness
把任务顺序、上下文、检查点、角色和恢复路径串起来。

核心变化

你不再把所有希望，塞进一句提示词。

你开始给 AI 一个能工作的环境：目标说清，边界写明，产物可检查，错误能恢复，经验能沉淀。

模型会继续变强。

一部分拆任务、调用工具、修复错误、整理上下文的工作，会越来越自动化。你未必每次都要手动做。

但关键不会消失

项目到底要去哪里，仍然需要你判断。

什么算做对，哪些东西不能动，什么风险可以接受，什么时候该继续，什么时候该停。这些判断，不能长期外包。

模型越方便，你越要保住自己的思考肌肉。

否则你只会看到一个看似完整的结果，却看不见它为什么成功，为什么失败。

这本书的练习方向

不是把你训练成术语很多的人。

把画面写成任务 让“好看、完整、顺手”变成能观察的验收项。
把任务拆成链条 让项目隔天回来还能继续，而不是重新猜一遍。
给角色划清边界 让 Agent 知道负责什么，也知道不能碰什么。
把经验沉淀成方法 让 Skill 减少重复交代，也减少每次执行的漂移。

流程的分寸

越到后面，越要学会不用。

小任务也许只需要一张想法落地卡；中等任务需要任务链和状态文件；责任真正分化时才需要 Agent；反复出现的做法，才值得沉淀成 Skill。

流程是为了保护直觉，不是替代直觉。

Vibe Coding 的珍贵之处，正是那个一开始模糊但真实的感觉。

结语

会用 AI 的人，是能看见任务结构的人。

模型会变，工具会换。目标要说清，边界要写明，产物要检查，错误要恢复，经验要沉淀。这些东西，不太会过期。
